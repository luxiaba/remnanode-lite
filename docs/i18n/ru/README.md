Здесь собраны руководства по развёртыванию и эксплуатации Remnanode Lite. Документы для разработчиков, сопровождения контракта и выпуска пока доступны только на английском; такие ссылки помечены отдельно.

## Быстрый маршрут для оператора

### Развернуть новый узел

1. Прочитайте [обзор проекта](project.md), чтобы проверить границы поддержки и нецели.
2. Выберите [Docker Compose](deployment-docker.md) либо [нативную установку systemd/OpenRC](deployment-native.md).
3. Заполните параметры и Secret по [справочнику конфигурации](configuration.md).
4. Перед выбором образа изучите [политику версий и тегов](versioning.md), особенно различия между `latest`, точной версией, `edge` и `sha-*`.
5. После запуска проверьте состояние health, связь с Panel и журналы rw-core по [руководству по эксплуатации](operations.md).

На хосте с `512 MiB RAM / 1 vCPU / 2 GB disk` сохраняйте заданные проектом лимиты памяти, CPU, PID, tmpfs и журналов. Не собирайте проект из исходного кода на таком рабочем узле.

### Обслуживать работающие узлы

1. Начните с healthcheck, журналов, обновления, отката и диагностики в [руководстве по эксплуатации](operations.md).
2. Проверьте источник и приоритет параметров в [справочнике конфигурации](configuration.md).
3. Для памяти, диска, журналов и времени остановки используйте [бюджет ресурсов (англ.)](../../development/resource-budget.md).
4. При проблемах протокола или жизненного цикла смотрите [архитектуру (англ.)](../../architecture.md) и [контракт 2.8.0 (англ.)](../../development/contract-2.8.0.md).
5. Для отката используйте ранее записанную точную версию или manifest digest. Не используйте `edge` и не полагайтесь на прежнее значение `latest`.

## Ключевые понятия

### Версия сборки и версия контракта — разные вещи

`Version` обозначает версию сборки Remnanode Lite, а `ContractVersion` — версию поведения официального Node, которую эта сборка реализует и сообщает Panel. Они могут меняться независимо: разработка новой линии `rnl.N` не означает поддержку ещё не реализованного официального контракта. Подробности приведены в [политике версий](versioning.md).

### Образ-кандидат — ещё не выпуск

`edge` и `sha-*` — сборки из `main`, но не официальные выпуски. Версия становится официальным выпуском только после публикации Git tag, GitHub Release и точного тега GHCR. Если нужно зафиксировать конкретный образ, используйте digest многоархитектурного манифеста.

### Совместимость проверяется на нескольких уровнях

Контрактные тесты, успешное подключение к Panel, ресурсные испытания и проверка дистрибутива отвечают на разные вопросы. Ни один из этих результатов не доказывает всё сразу, поэтому всегда указывайте, что именно было проверено.

## Термины

| Термин | Значение |
| --- | --- |
| Node | Долгоживущий процесс `remnanode-lite`, принимающий запросы Panel и владеющий жизненным циклом rw-core |
| rw-core | Бинарный Xray Core, обслуживающий proxy traffic и управляемый Node |
| `Version` | Идентичность сборки проекта, GitHub Release и точного тега образа |
| `ContractVersion` | Реализованный официальный behavioral baseline, по умолчанию сообщаемый Panel |
| operation epoch | Возрастающий token владения одной операцией Xray start/stop, не идентичность процесса |
| process lease | Краткоживущий допуск, привязанный к process epoch rw-core и abstract socket, не позволяющий mutation перейти между процессами |
| lifecycle lease | Shared/exclusive координация HTTP для start, stop, plugin/user mutation и reset-capable stats; не lock-файл |
| кандидат на выпуск | Текущий коммит `main`, представленный неизменяемым образом `sha-<commit>` |
| manifest digest | Контентный адрес `sha256:...` мультиархитектурного GHCR index, надежнее перемещаемого тега |

## Сопровождение перевода

- Сначала обновляется английский исходный документ, затем перевод.
- Команды, пути, имена переменных, числовые лимиты и support claims не локализуются по смыслу.
- Реальные Secrets, JWT, сертификаты, адреса хостов и пользовательские данные запрещены в примерах.
- Исторические аудиты и roadmap не заменяют код и автоматизацию как источник текущего поведения.
- При расхождении следуйте английскому источнику и обновляйте metadata SHA-256 по [политике локализации (на английском)](../README.md).
